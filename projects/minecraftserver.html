PHP MySQL CRUD

Displayed here is the code for two separate scripts. One operates as the display of the entries and the other as an interface to the database. The original intent of the script was to provide a client with an easy interface to updating the news ticker on their front page. The user can add, delete, and edit news entries.

I opted to use a functional programming approach rather than OOP as it is easier to maintain for the purposes of being a simple news updater. The benefits of using an OOP approach would be the use of prepared statements which would provide more security. The display code can easiy be embedded into an existing template/webpage and styled using CSS to fit with the theme.

One of the shortcomings of this code is the inefficient use of $_GET to pass delete and edit information to itself. Using session would have been cleaner and more efficient. Using $_GET makes it simple to debug in the URL as the data can be manipulated without viewing the source code. However passing information in the URL does make it less secure, the specific application of this script did not require a high level of security.
